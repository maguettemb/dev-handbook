# 5. CI/CD

Automatiser la vérification et la livraison du code : de "je pousse mon code" à "le code validé arrive en production sans intervention manuelle".

## Ce que couvre cette section

- **[Concepts de base](concepts.md)** — CI vs CD, ce qu'un pipeline vérifie
- **[Pipeline type](pipeline-type.md)** — lint → test → build → deploy, avec des exemples GitLab CI / GitHub Actions

## Contexte

Cette section s'appuie directement sur le projet [DevOps Copilot](../journal.md), qui expose des outils GitLab (dont `get_pipeline_status`) à Claude Code — l'occasion de comprendre les pipelines de l'intérieur.
