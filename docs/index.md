# Dev Handbook

Bienvenue dans mon carnet de route pour la transition **Data Scientist → Software Engineer**.

Ce site n'est pas une documentation figée : il se remplit **au fur et à mesure** d'un projet réel (voir [Journal de bord](journal.md)). Chaque page correspond à une pratique testée en conditions réelles, pas juste théorique.

## Pourquoi ce handbook

En tant que data scientist, l'essentiel de mon travail tournait autour de notebooks, d'expérimentation et de modèles. Le passage au software engineering demande d'autres réflexes : structure de projet, tests, CI/CD, déploiement, travail en équipe sur du code partagé.

Ce site documente cette montée en compétences, section par section, avec :

- Ce que j'ai appris
- Ce qui m'a posé problème
- Les outils comparés avec leurs pour/contre
- Des exemples concrets tirés du projet fil rouge
